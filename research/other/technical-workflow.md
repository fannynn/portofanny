# Technical Workflow I Used

This project also required me to use more technical tools than I expected at first.

I used:

- `Cursor IDE` as my main working environment
- `Git` and `GitHub` for version control
- `Git Bash` as one of the terminals I used during setup and repository work
- `Python` for transcript collection
- `youtube-transcript-api` to pull YouTube transcripts
- `Node.js` for additional environment setup

I also worked with a custom AI setup:

- I used my own `SumoPod` API key
- Because the workflow has to be conducted mostly in manual, I installed `WSL`
- I used the `WSL` terminal inside Cursor IDE
- I configured SumoPod-related environment variables manually there
- I installed Claude Code manually through the terminal
- I tested manual model configuration for `claude-sonnet-4-6`

I also used `Codex` for many of the fixes when I still had unresolved terminal issues. That was especially helpful when I got stuck on setup problems, file structure issues, transcript collection workflow, and Git-related confusion.
